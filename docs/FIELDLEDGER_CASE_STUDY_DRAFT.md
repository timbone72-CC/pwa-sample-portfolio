# FieldLedger Case Study Draft

## 1. Purpose

FieldLedger is the flagship proof-of-work for the PWA Sample Portfolio.

It shows that a lightweight, mobile-friendly, offline-first business app can solve a real workflow without requiring a full enterprise system.

## 2. Plain-Language Description

FieldLedger is a mobile-friendly work tracking app for 1099 field workers.

It helps a worker track jobs, expenses, mileage, pay period totals, tax estimate details, reports, backups, exports, and optional Google Sheets handoff.

## 3. What FieldLedger Proves

FieldLedger proves the builder can create:

- lightweight offline-first PWAs
- mobile-friendly business workflows
- local browser data storage
- job tracking records
- expense tracking records
- mileage tracking records
- printable reports
- CSV exports
- JSON backup and restore
- trusted-user setup instructions
- optional Google Sheets workflow support
- practical small-business tools without enterprise bloat

## 4. Core Working Features

Current working feature areas:

- dashboard-style app navigation
- Bucking job tracking
- Torque Turn job tracking
- expense tracking
- mileage tracking
- saved records lists
- pay period summary
- tax estimate breakdown
- JSON backup download
- JSON restore/import
- CSV export
- printable full report
- printable timesheet report
- ticket and receipt photo attachment support
- offline/PWA install support
- GitHub Pages deployment
- optional Trusted Sheet send workflow

## 5. Business Value

FieldLedger shows how a small-business app can replace scattered notes, manual math, screenshots, paper records, and repeated spreadsheet cleanup.

It is built for practical use:

- fast entry on a phone
- readable summaries
- backup before destructive actions
- exportable records
- printable reports
- optional handoff to office review

## 6. Portfolio Message

Public-facing message:

FieldLedger is a real working example of a lightweight business PWA built for field work.

It demonstrates the kind of tools that can be adapted into job trackers, timesheets, invoice helpers, field-work logs, reporting tools, and Google Sheets workflows.

## 7. What Should Stay Private

Do not publicly expose:

- unfinished internal planning
- trusted-user private setup details
- import tokens
- private Sheet URLs
- rough notes
- client-style internal contracts
- experimental ideas not ready for public viewing

## 8. Reusable Ideas for Future Samples

FieldLedger can inspire reusable sample-app patterns:

- mobile PWA shell
- saved record cards
- export/backup panel
- printable reports
- future add-on cards
- Google Sheets handoff as an optional module
- local-first business workflow design

## 9. Current Decision

FieldLedger should be used as the flagship case study while the generic Job Tracker PWA becomes the first simplified public sample app.
